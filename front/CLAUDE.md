# front/ — Streamlit UI

## 역할 범위

- 사용자 자연어 입력 수신
- 결제자(payer) 이름 입력 수신 (텍스트 영역 하단)
- ai/ LangGraph 워크플로우를 직접 import하여 실행 (`graph.invoke(state)`)
- 채팅 인터페이스로 정산 결과 출력 (최신 결과가 상단)
- 복사 가능한 공유용 정산 메시지 표시

> front/, ai/, calculator/ 모두 같은 Python 프로세스에서 실행된다. HTTP 통신 없음.

## 화면 구조

```
[좌측 사이드바]              [메인 영역]
  📋 상황별 예시              # 💸 AI 정산 비서
  💳 대화 초기화              ─────────────────────
  ─────────                  👤 첫 번째 질문
  🍺 예시1 [채우기↗]          🤖 첫 번째 결과
  💳 예시2 [채우기↗]          ─────────────────────
  ...                        👤 두 번째 질문 (아래로 쌓임)
                             🤖 두 번째 결과
                             ─────────────────────
                             [정산 상황 텍스트 영역 ]
                             [결제자 이름 ] [전송 →]
```

메시지는 오래된 것부터 위→아래 순서로 표시. 전송 후 자동으로 하단 스크롤.

## 세션 상태

```python
st.session_state.messages  # 대화 이력 (user/assistant 메시지 리스트)
st.session_state["draft"]  # 입력창 텍스트 (사이드바 예시로 채울 수 있음)
st.session_state["_payer"] # 결제자 이름 입력값
```

- `messages` 각 항목: `{"role": "user"/"assistant", "content"/"결과필드들"..., "payer": str}`
- draft 클리어는 `_clear_draft` 플래그 패턴 사용 (위젯 렌더 전에 처리)

## 화면 흐름

1. 정산 상황을 텍스트 영역에 입력 (좌측 예시 클릭 시 채워짐)
2. 결제자 이름 입력 (선택, 없으면 개별 부담액 표시)
3. `전송 →` 클릭 → AI 분석 → 결과가 채팅 히스토리 상단에 추가
4. 추가 조건을 다시 입력하면 자동으로 피드백(재계산) 모드로 전환
5. 공유 메시지 코드 블록 우상단 복사 버튼 제공

## 피드백 모드 자동 전환

`_invoke_graph()` 내부에서 판단:
- `messages`에 이전 assistant 메시지가 있고 `parsed_json.participants`가 존재 → 피드백 모드
- 없으면 → 초기 정산 모드
- 별도 "재계산" 버튼 없음

## 결과 표시 분기

- `payer` 있음 → "B → A: 23,750원" 형태로 이체 금액 표시
- `payer` 없음 → 개인 부담액 카드 표시
